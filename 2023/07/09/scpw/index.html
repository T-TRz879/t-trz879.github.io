一、概述

在Linux可以使用scp命令服务器A的资源发送到服务器B上，这个命令确实很方便，对于我来说并不是特别方便（我是很喜欢偷懒的人，所以就想办法能不能把偷懒做到极致，这时候就需要“造个轮子”）。

1.1、需求概述

我需要将服务打包后的文件发送到很多个服务器中，我需要打开多个terminal传输包，或者等上一个传输结束后再次传输，如果这个能一个命令解决且是并行的那该多好啊。如果能做到配置一次服务器的信息，每次只需要选择需要传输的选项的话，就不用输入服务器的信息和密码了。

总结：

  • 多个传输任务并行执行
  • 减少用户输入（配置化）

二、方案

2.1、包装scp

这个方案很简单，在goroutine中调用系统的scp命令。这也能完成目标，但并不能从造轮子的过程中学习到什么。很显然不符合我个人的要求。

2.2、实现scp

scp命令是Linux下的一个基于SSH协议的文件传输命令，它用于在本地主机和远程主机之间复制文件。scp命令是安全的，因为它在传输数据时使用SSH协议提供的强大的加密和认证机制。

（1）scp中内部逻辑

  1. 使用SSH客户端发送scp命令后，需要自行组织和维护传输给服务器的命令和数据内容。
  2. 发送方：每次在发送数据或发送命令后，服务端都会返回状态码，客户端需要检测状态码才能决定接下来行为。
  3. 接受方：每次在接受数据或接受命令后，自身需要判断用户是否关闭、退出本次操作，需要返回给客户端状态码，这样客户端会判断状态码决定是否关闭资源还是继续传输。

注意：在实现scp的过程中少检测和多检测状态码都会让程序陷入永远阻塞状态，在刚开始实现过程中建议将每次发送命令和接受状态码都打印日志，以便调试和发现问题

（2）scp内部参数

  1. -f：表示从远程系统获取文件。当你在本地系统上运行scp命令复制远程文件时，远程系统上的scp进程会以-f选项运行。
  2. -t：表示向远程系统传输文件。当你在本地系统上运行scp命令复制文件到远程系统时，远程系统上的scp进程会以-t选项运行。
  3. -d：表示目标是一个目录。当复制多个文件或者一个目录时，scp会以-d选项运行，这告诉scp应该在目标系统上创建一个目录来存放复制的文件。
  4. -r：表示递归复制。当复制一个目录及其所有子目录和文件时，scp会以-r选项运行。
  5. -p：表示保留文件的修改/访问时间和权限。当以-p选项运行时，scp会尝试在目标文件上设置与源文件相同的修改/访问时间和权限。
  6. -l：限制使用的带宽，以 Kbit/s 为单位。

（3）scp内部命令

scp在底层使用了一套简单的文本协议来控制文件和目录的复制操作。这个协议包含一些命令，这些命令是在scp的源代码中定义的，并以单个字符的形式发送。这些命令用来传输文件名、时间、文件类型和是否结束等。

  1. T：设置文件的访问时间和修改时间。此命令后面跟随两个以空格分隔的数值，分别表示访问时间和修改时间。这些时间值以UNIX时间戳的形式表示，即从1970年1月1日开始的秒数。这个命令是由源系统发送的，以告诉目标系统应该如何设置复制文件的时间属性。

    1. atime：文件的最后访问时间，以1970年1月1日开始的秒数表示。
    2. atime_usec：文件的最后访问时间的微秒部分。
    3. mtime：文件的最后修改时间，以1970年1月1日开始的秒数表示。
    4. mtime_usec：文件的最后修改时间的微秒部分。

    格式：T atime atime_usec mtime mtime_usec，例如：T1688885748 0 1688885800 0（微秒部分一般都是0）

  2. C：开始复制一个文件。此命令后面跟随文件的权限，文件的大小和文件的名字，这些值都以空格分隔。权限值是一个四位的八进制数，表示UNIX的文件权限。大小值是文件的字节数。名字是文件的名字，不包含路径。这个命令是由源系统发送的，以告诉目标系统要复制的文件的信息。

    1. mode：文件的权限，以四位的八进制数表示。
    2. length：文件的长度，以字节为单位。
    3. name：文件的名字。

    格式：Cmode length name，例如：C0664 6 hello.txt（文件名为hello.txt，长度为6，权限为0664）

  3. D：开始复制一个目录。此命令的语法与C命令相同，但是表示的是一个目录，而不是一个文件。这个命令是由源系统发送的，以告诉目标系统要复制的目录的信息。
    格式：Dmode length name，例如：D0755 0 dir（文件名为dir，长度为0，权限为0755）

  4. E：结束复制一个目录。此命令没有参数。当源系统完成一个目录的复制后，它会发送这个命令来告诉目标系统可以关闭这个目录。这个命令也用于标志整个复制操作的结束。

（4）scp内部响应码

  1. 0：成功。这个响应码表示前面的操作已经成功完成。
  2. 1：错误。这个响应码表示前面的操作失败了，后面会跟随一行文本，解释了发生了什么错误。
  3. 2：致命错误。这个响应码表示发生了一个严重的错误，scp进程无法继续执行。后面会跟随一行文本，解释了发生了什么错误。
