Read流程

基础流程

  1. 如果开启了加密和压缩功能，JuiceFS将一个Block先解压在解密放入内存中。如果未开启则会跳过解压和解密过程
  2. Kernel Fuse读取JuiceFS数据时，现在JuiceFS的缓存中查看是否由此数据，没有JuiceFS则会去Strorage中获取
  3. JuiceFS将其切分成每段数据128KB，将其以128KB每份发送到Kernel中，Kernel将数组组合成1MB发送到Application

Read方法调用流程

  1. vfs将off（开始读取索引）和buf（数据存放的字节数组）传入Read方法

    注意：buf数组的长度有一个适应大文件的过程，buf一开始的大小为16KB，如果一开始开128KB对于一个小文件来说很是浪费。buf会经历四个过程：16KB - > 32KB -> 64KB -> 128KB。这样小文件即不会浪费空间，大文件也会逐渐提升buf长度。

  2. 在fileReader.read中，小文件或最后取的一段数据小于32KB则会去缓存中寻找处于哪个Block中

VFS.Read

func (v *VFS) Read(ctx Context, ino Ino, buf []byte, off uint64, fh uint64) (n int, err syscall.Errno) {
   size := uint32(len(buf))
   if IsSpecialNode(ino) {
      if ino == logInode {
         n = readAccessLog(fh, buf)
      } else {
         h := v.findHandle(ino, fh)
         if h == nil {
            err = syscall.EBADF
            return
         }
         data := h.data
         if off < h.off {
            data = nil
         } else {
            off -= h.off
         }
         if int(off) < len(data) {
            data = data[off:]
            if int(size) < len(data) {
               data = data[:size]
            }
            n = copy(buf, data)
         }
         if len(h.data) > 2<<20 {
            // drop first part to avoid OOM
            h.off += 1 << 20
            h.data = h.data[1<<20:]
         }
      }
      return
   }

   defer func() {
      readSizeHistogram.Observe(float64(n))
      logit(ctx, "read (%d,%d,%d): %s (%d)", ino, size, off, strerr(err), n)
   }()
   h := v.findHandle(ino, fh)
   if h == nil {
      err = syscall.EBADF
      return
   }
   if off >= maxFileSize || off+uint64(size) >= maxFileSize {
      err = syscall.EFBIG
      return
   }
   if h.reader == nil {
      err = syscall.EACCES
      return
   }
   if !h.Rlock(ctx) {
      err = syscall.EINTR
      return
   }
   defer h.Runlock()
    // 防止read时，还有数据没有写入Block
   _ = v.writer.Flush(ctx, ino)
    // 具体读取方法
   n, err = h.reader.Read(ctx, off, buf)
   for err == syscall.EAGAIN {
       // 读取失败，错误码为syscall.EAGAIN 则再一次读取
      n, err = h.reader.Read(ctx, off, buf)
   }
   if err == syscall.ENOENT {
      err = syscall.EBADF
   }
   h.removeOp(ctx)
   return
}

fileReader.Read

  1. 用size记录buf的长度；开始读取索引已经到file的末尾或当前size为0，则退出

  2. 用block记录offset和size，在block中：offset = off，len = size

  3. block.end() = block.off + block.len，代表读取到end()结束。block.end() > 文件长度，将block.len修改为文件长度 - block.off

  4. 最后一次读取没有32KB或文件本身没有32KB时，会调用readAhead方法，本次需要读取内容很有可能在上一次的末尾

  5. 调用splitRange查看block在哪个范围内，主要有两种情况：

    1. off和off+len在同一个Block
    2. off和off+len不在同一个Block

  6. 调用prepareRequests对ranges范围进行查找，查看ranges存在在哪几个Slice中

func (f *fileReader) Read(ctx meta.Context, offset uint64, buf []byte) (int, syscall.Errno) {
   f.Lock()
   defer f.Unlock()
   f.acquire()
   defer f.release()
   // len(buf) 16KB -> 32KB -> 64 KB -> 128KB
   if f.err != 0 || f.closing {
      return 0, f.err
   }
   // 获取buf的大小
   size := uint64(len(buf))
   if offset >= f.length || size == 0 {
      return 0, 0
   }
   // block.end() = offset + size
   block := &frange{offset, size}
   // 当前读完后的长度block.end() > 文件长度则需要设置block.len为f.length - block.off
   if block.end() > f.length {
      block.len = f.length - block.off
   }
   //logger.Warnf("offset:%d buf.len:%d file.size:%d", offset, len(buf), f.length)
   f.cleanupRequests(block)
   // lastBS 32KB is cache block size
   var lastBS uint64 = 32 << 10
   if block.off+lastBS > f.length {
      // last block cache is 32KB
      lastblock := frange{f.length - lastBS, lastBS}
      // check file length less than 32KB
      if f.length < lastBS {
         lastblock = frange{0, f.length}
      }
      // read from cache
      f.readAhead(&lastblock)
   }
   ranges := f.splitRange(block)
   reqs := f.prepareRequests(ranges)
   defer func() {
      for _, req := range reqs {
         s := req.s
         s.refs--
         if s.refs == 0 && s.state == INVALID {
            s.delete()
         }
      }
   }()
   f.checkReadahead(block)
   return f.waitForIO(ctx, reqs, buf)
}

fileReader.splitRange

  1. 将block.off和block.end()放入ranges中，ranges代表当前读取的开始索引和结束索引
  2. 匿名func的作用：判断当前p是否已经存在于ranges中，存在则返回true，不存在则返回false
  3. ranges的长度和参数意义说明：
    1. ranges最长3，最少2
    2. ranges长度为2时，ranges[0]为block.off，ranges[1]为block.end()
    3. ranges长度为3时（未经过排序），ranges[0]为block.off，ranges[1]为block.end()，ranges[2]为block.off和block.end()之间整数倍的4MB
    4. ranges用来解决跨Block读取内容。off和end()跨Block时，ranges则会添加Block之间的分割索引
  4. 最后对ranges进行升序
ranges不同时段的日志
  • 开始Read时，开始时ranges[1]和ranges[0]的跨度是16KB -> 32KB -> 64 KB -> 128KB的递增

  • 跨片时，会在ranges[2]位置上添加第一个Block结尾索引

// 对block进行拆分，block.off和block.end()分别位于什么区间内
func (f *fileReader) splitRange(block *frange) []uint64 {
    ranges := []uint64{block.off, block.end()}
    // lambda func
    contain := func(p uint64) bool {
    // 这个方法用来判断p是否已经在ranges中。
    //不在ranges中则在下面visit方法中添加入ranges，在ranges中无需添加
        for _, i := range ranges {
            if i == p {
                return true
            }
        }
        return false
    }
    f.visit(func(s *sliceReader) {
        if s.state.valid() {
            // s.block.off在block中 且 ranges中没有s.block.off
            if block.contain(s.block.off) && !contain(s.block.off) {
                ranges = append(ranges, s.block.off)
            }
            // s.block.end()在block中 且 ranges中没有s.block.end()
            if block.contain(s.block.end()) && !contain(s.block.end()) {
                ranges = append(ranges, s.block.end())
            }
        }
    })
    // 对ranges升序排列
    sort.Slice(ranges, func(i, j int) bool {
        return ranges[i] < ranges[j]
    })
    return ranges
}

fileReader.newSlice

​ 使用newSlice的地方有两处：

  • readAhead中
  • prepareRequests中。第一次的16KB的buf会创建第一个[0,16KB]的Slice

  1. s.indx代表当前block.off位于第几个Chunk中

  2. blockend代表当前block.off位于以blockend长度结尾的Block，可参考代码中的示例

  3. 校验当前s.block.end()和文件长度、blockend的关系（s.block.end() = s.block.off + s.block.len）

    1. s.block.end() > f.length，即[s.block.end(),f.length]是多余的，只需要[s.block.off,f.length - s.block.off]

    2. s.block.end() > blockend，即已经涉及跨片，只选择s.block.off所在的Block，也就是[s.block.off,blockend - s.block.off]

  4. 更新当前block的off和len。block.off设置为s.block.end()，s.block.end()为本次实际需要读取的末尾索引；block.len减去s.block.len（根据上面上个判断更更新s.block.len来说，s.block.len永远小于或等于block.len），用于下一个Block

  5. 创建s.block.len长度的byte数组，用来存放读取内容

  6. 将fileWriter的最后一个节点（fileWriter中用last存放）设置为当前新建sliceReader的prev节点

  7. 将slieWriter设置为fileWriter的最后一个节点

  8. 另一个线程调用run()方法完成读取Meta和数据

  9. 如下是newSlice创建过程的日志，除了前两次和最后一次之外的block.len都是4MB

    1. 第一次是16KB的读取buf空间，需要创建Slice
    2. 第二次len需要从16KB开始到4MB结束，len=4MB-16KB
    3. 最后一次的道理和第二次一样，len=file.length-block.off

// 当前的Block属于新的Slice
func (f *fileReader) newSlice(block *frange) *sliceReader {
   //logger.Warnf("new slice")
   s := &sliceReader{}
   s.file = f
   s.lastAccess = time.Now()
   // s.indx当前的off位于第几个Chunk
   s.indx = uint32(block.off / meta.ChunkSize)
   s.block = &frange{block.off, block.len}
   // random read
   // 当前off位于的Block的结束
   // off                0MB       3.9MB          4MB
   // blockend        4MB       4MB       8MB
   blockend := (block.off/f.r.blockSize + 1) * f.r.blockSize
   // 当前block.end是否超过文件总大小
   if s.block.end() > f.length {
      s.block.len = f.length - s.block.off
   }
    // 当前block.end()是否超过blockend,超过则更新block.len
   if s.block.end() > blockend {
      s.block.len = blockend - s.block.off
   }
   // 更新block的off和len用于下一个block
   block.off = s.block.end()
   block.len -= s.block.len
   // 开 s.block.len个存储空间
   s.page = chunk.NewOffPage(int(s.block.len))
   s.cond = utils.NewCond(&f.Mutex)
   // 将当前的s前一个设置为f的最后一个
   s.prev = f.last
   // f的最后一个节点更新为当前s
   *(f.last) = s
   f.last = &(s.next)
   logger.Warnf("newSlice block:[%d,%d]", s.block.off, s.block.len)
   // get data things
   go s.run()
   atomic.AddInt64(&readBufferUsed, int64(s.block.len))
   return s
}

fileReader.prepareRequests

func (f *fileReader) prepareRequests(ranges []uint64) []*req {
   var reqs []*req
   edges := len(ranges)
   for i := 0; i < edges-1; i++ {
      var added bool
      // b.off是本次开始读 b.len是本次读多少
      b := frange{ranges[i], ranges[i+1] - ranges[i]}
      f.visit(func(s *sliceReader) {
         if !added && s.state.valid() && s.block.include(&b) {
            s.refs++
            s.lastAccess = time.Now()
            reqs = append(reqs, &req{frange{ranges[i] - s.block.off, b.len}, s})
            added = true
         }
      })
      // 第一次读会走下面这个
      if !added {
         for b.len > 0 {
            logger.Warnf("prepareRequests create slice")
            s := f.newSlice(&b)
            s.refs++
            reqs = append(reqs, &req{frange{0, s.block.len}, s})
            //logger.Warnf("b.off:%d b.len:%d s.block.len:%d", b.off, b.len, s.block.len)
         }
      }
   }
   return reqs
}

fileReader.readAhead

func (f *fileReader) readAhead(block *frange) {
   f.visit(func(r *sliceReader) {
      if r.state.valid() && r.block.off <= block.off && r.block.end() > block.off {
         if r.state == READY && block.len > f.r.blockSize && r.block.off == block.off && r.block.off%f.r.blockSize == 0 {
            // next block is ready, reduce readahead by a block
            block.len -= f.r.blockSize / 2
         }
         if r.block.end() <= block.end() {
            block.len = block.end() - r.block.end()
         } else {
            block.len = 0
         }
         block.off = r.block.end()
      }
   })
   if block.len > 0 && block.off < f.length && uint64(atomic.LoadInt64(&readBufferUsed)) < f.r.readAheadTotal {
      if block.len < f.r.blockSize {
         block.len += f.r.blockSize - block.end()%f.r.blockSize // align to end of a block
      }
      logger.Warnf("readAhead")
      f.newSlice(block)
      if block.len > 0 {
         f.readAhead(block)
      }
   }
}

sliceReader.run

func (s *sliceReader) run() {
    f := s.file
    f.Lock()
    defer f.Unlock()
    if s.state != NEW || f.shouldStop() {
        s.done(0, 0)
    }
    s.state = BUSY
    // 第几个Chunk
    indx := s.indx
    // inode号
    inode := f.inode
    f.Unlock()

    f.Lock()
    // 文件大小
    length := f.length
    f.Unlock()
    var chunks []meta.Slice
    // it is read metadata info
    err := f.r.m.Read(meta.Background, inode, indx, &chunks)
    f.Lock()
    if s.state != BUSY || f.err != 0 || f.closing {
        s.done(0, 0)
    }
    if err == syscall.ENOENT {
        s.done(err, 0)
    } else if err != 0 {
        f.tried++
        trycnt := f.tried
        if trycnt >= f.r.maxRetries {
            s.done(syscall.EIO, 0)
        } else {
            s.done(0, retry_time(trycnt))
        }
    }

    s.currentPos = 0
    if s.block.off > length {
        s.block.len = 0
        s.state = READY
        s.done(0, 0)
    } else if s.block.end() > length {
        s.block.len = length - s.block.off
    }
    need := s.block.len
    f.Unlock()

    p := s.page.Slice(0, int(need))
    defer p.Release()
    var n int
    ctx := context.TODO()
    // it is really read bytes
    // (uint32(s.block.off))%meta.ChunkSize 决定在该Chunk的哪个位置开始读
    n = f.r.Read(ctx, p, chunks, (uint32(s.block.off))%meta.ChunkSize)

    f.Lock()
    if s.state != BUSY || f.shouldStop() {
        s.done(0, 0)
    }
    if n == int(need) {
        s.state = READY
        s.currentPos = uint32(n)
        s.file.tried = 0
        s.lastAccess = time.Now()
        s.done(0, 0)
    } else {
        s.currentPos = 0 // start again from beginning
        err = syscall.EIO
        f.tried++
        _ = f.r.m.InvalidateChunkCache(meta.Background, inode, indx)
        if f.tried >= f.r.maxRetries {
            s.done(err, 0)
        } else {
            s.done(0, retry_time(f.tried))
        }
    }
}

redisMeta.Read

  1. 判断当前inode的index的Chunk是否已经读取过，读取过则返回
  2. 通过inode和indx在redis中寻找具体存储信息
  3. 调用buildSlice方法，合并需要的Block
  4. 将合并后的Block缓存在缓存文件中
func (r *redisMeta) Read(ctx Context, inode Ino, indx uint32, chunks *[]Slice) syscall.Errno {
    f := r.of.find(inode)
    if f != nil {
        f.RLock()
        defer f.RUnlock()
    }
    // 判断当前inode的第indxChunk是否已经有
    if cs, ok := r.of.ReadChunk(inode, indx); ok {
        *chunks = cs
        return 0
    }
    defer timeit(time.Now())
    //从redis中获取inode和indx的value,从而利用反序列的方式读取出来
    vals, err := r.rdb.LRange(ctx, r.chunkKey(inode, indx), 0, 1000000).Result()
    if err != nil {
        return errno(err)
    }
    ss := readSlices(vals)
    *chunks = buildSlice(ss)
    // 缓存下来
    r.of.CacheChunk(inode, indx, *chunks)
    if !r.conf.ReadOnly && (len(vals) >= 5 || len(*chunks) >= 5) {
        go r.compactChunk(inode, indx, false)
    }
    return 0
}

buildSlice详解

什么是Chunk、Slice和Block？

​ 文件在JuiceFS先被分为固定大小的Chunk「固定上限为64MB」。Slice是Chunk的一小部分，没有固定大小。Block是Slice的一小部分，一个Slice可以被切成多个Block「默认大小为4MB」，Block是JuiceFS中具体存储的物理object。

​ Slice的cut和visit方法让JuiceFS具备随机写的能力。cut方法是将多个Slice列表按照chunkId升序排列，chunkId大的Slice覆盖chunkId小的Slice范围。visit方法在Slice列表cut后，将其Slice列表从前往后组合成需要获取每个chunkId的Slice的数据范围。

Slice结构

属性 解释
uint64 ChunkId Chunk id
uint32 Size Slice的大小
uint32 Off Slice开始索引
uint32 Len Slice需读取长度

注意：slice是cut和visit使用的，Slice是visit对slice处理的结果集合

buildSlice

​ slice是按照chunkId进行升序的slice数组，需要对其进行cut操作。当遍历到i时，slice[i]会覆盖slice[0] - slice[i - 1]与slice[i]的区间，换句话说：slice[i]的区间是每次必须要的如果前面也出现此区间则会覆盖之前的区间。

​ 用root来记录当前已经处理的链表节点「第一次循环时，将s赋给root」。

​ 对slice进行cut后，对root节点的双向链表进行遍历，获取到从开始到结束的Slice数组，便于寻找目标数据。

buildSlice过程

  1. 根据chunkId对sliceList进行升序，遍历sliceList的过程中使用root来记录当前已经切分好的分片序列
  2. root是一个双向链表，记录着已经切片完成的slice序列
  3. 遍历sliceList过程中，本次循环中的slice的[off,off+len]数据是需要覆盖root链表的区间
  4. 每次遍历sliceList时，先对s.pos左边切分，在对s.pos + s.len进行右切分（[pos,pos+len]是必须要的，所以不用切分）
    1. 每次调用cut会返回
    2. 对pos进行左切分后得到的，left是s.pos左边已经切分好的节点，需将left设置为s.left。
    3. right需要作为root节点对pos右边进行右切分，右切分返回的，我们将其right设置为s.right节点
  5. 在切片过程中，off和len是随着切片的细化而变化，但pos和size不会变，分别是slice在Chunk中的下标和slice原本的大小。

visit过程

Slice vist是将slice列表merge和cut后的结果整合成我们需要获取的数据集合。其中将结果放入Slice集合中，Slice的存储结构为[ChunkId,Size,Off,Len]。

在slice merge和cut后，root指向最后一个chunkId的cut片对象，root都有可能在链表的头、中和尾，我们先对root向左递归进行visit，在向其右递归进行visit。

递归中所做的事情如下：

  1. 用pos记录当前处理到的下标，Slice[] chunk用来记录返回数据，s为当前的节点
  2. s.pos > pos（当前节点的pos和pos之间是缺少数据的），用当前s的[0,pos]数据来填充这段Slice，将Slice放入chunk中，pos移动到s.pos
  3. 添加当前s到chunk中，pos += s.len（s.len代表当前s切片后的实际需要大小）
  4. 在处理过程中遇到slice片之间断层时，我们将其设置为[0,0,0,len]的一个Slice，这样在真实获取数据时，可以根据Size和Off来标识当前Slice为一个断层Slice
func buildSlice(ss []*slice) []Slice {
  // 用root记录当前处理已经cut好的slice；第一次循环只会将root设置为s
  // root实则是一个双向链表；每次遍历过程中，s的内容是必须要的，主要是对s.pos在root中的左边位置进行cut和对s.pos+s.len在root中右边位置进行cut
  // cut方法有两个返回值，第一个是传入的pos与root切分的左边节点；第二个是由于root左边被切了，右边需要更新对应的开始索引，右边用于对s.pos + s.len进行右切分
  var root *slice
  for i := range ss {
    s := new(slice)
    *s = *ss[i]
    var right *slice
    // 以s.pos作为基准对root的[root的最左的节点的pos,pos]和[pos,root的最右的节点的pos + len]
    // 左边已经被切出来了，所以将当前s.left更新为切好的第一个返回值
    s.left, right = root.cut(s.pos)
    // 以s.pos + s.len作为基准对right「（已经左切分好的root）的右边」进行右切分，得到的以s.pos+s.len的root节点作为s.right「因为遍历的当前节点是需要全部取到的，所以不用管与s重合部分」
    _, s.right = right.cut(s.pos + s.len)
    //每次将s作为root「当作当前已经处理好的节点」
    root = s
  }
  // it is cur deal pos
  var pos uint32
  // it is cur deal chunk
  var chunk []Slice
  root.visit(func(s *slice) {
    // 处理断层现象s.pos和上一个pos之间缺少内容，ChunkId = 0来表示这个断层slice
    if s.pos > pos {
      chunk = append(chunk, Slice{Size: s.pos - pos, Len: s.pos - pos})
      pos = s.pos
    }
    // 每次将cut的Slice填充到slice中
    chunk = append(chunk, Slice{Chunkid: s.chunkid, Size: s.size, Off: s.off, Len: s.len})
    pos += s.len
  })
  return chunk
}

cut

​ slice cut方法根据当前节点slice对传入的pos进行切分（root代表切分完成的链表节点）。

​ 当前pos可能会覆盖到多个左slice或者多个右slice。当前节点所在的片为4，需要先对pos进行左切片；先寻找到pos所在的片，在2的基础上以pos为基准对其左右进行分割。

​ 对2左分割后得到pos左边的2和pos右边的right。再以right作为基础对pos+len进行右切割，会先从right寻找到pos+len所在的节点，在对pos+len进行右切分。

​ pos+len所在的节点为4，我们只需要4中pos+len的右部分。图中粉色部分都是被[pos,pos+len]覆盖的区间。

  1. 当slice.pos <= root.pos时会进行左递归找出root中最左与slice.pos重合的slice片段对其进行切分
  2. 当slice.pos >= root.pos时会进行右递归找出root中最右与slice.pos + slice.len重合的slice片段对其进行切分
  3. 当slice.pos在root的片段中当前片段中时，会将root当前片段中与slice重合的部分设置为slice，将root与slice边缘部分对root进行切分。
func (s *slice) cut(pos uint32) (left, right *slice) {
    if s == nil {
        return nil, nil
    }
    if pos <= s.pos {
        // pos 在当前s的左边节点中，需要对其进行左边寻找
        // 如果缺少slice则会创建新的slice来填充其余部分
        if s.left == nil {
            s.left = newSlice(pos, 0, 0, 0, s.pos-pos)
        }
        left, s.left = s.left.cut(pos)
        return left, s
    } else if pos < s.pos+s.len {
        // pos 就在当前s中，对s进行切割
        l := pos - s.pos
        right = newSlice(pos, s.chunkid, s.size, s.off+l, s.len-l)
        right.right = s.right
        s.len = l
        s.right = nil
        return s, right
    } else {
        // pos 在当前s的右边节点中，需要对其进行右边寻找
        // 如果缺少slice则会创建新的slice来填充其余部分
        if s.right == nil {
            s.right = newSlice(s.pos+s.len, 0, 0, 0, pos-s.pos-s.len)
        }
        s.right, right = s.right.cut(pos)
        return s, right
    }
}
